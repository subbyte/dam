# Supply-chain security

Last verified: 2026-06-11

Motivated by: operational policy, not an architectural decision ‚Äî no ADR.

| Dependency type | CVE defense | Zero-day defense |
|---|---|---|
| **npm (pnpm)** | **Mend SCA** via `cd.yml` ‚Äî scans lockfile for transitive CVEs, auto-files GitHub issues. **Dependabot** ‚Äî direct-dep advisories, auto-files PRs. **pnpm audit** ‚Äî runs on PRs and daily via `mise run scan`, checks the npm advisory database. Remediation: transitive ‚Üí scoped `pnpm.overrides`; direct ‚Üí bump in workspace `package.json`. | Lockfile pins exact versions (`--frozen-lockfile` in CI). `minimumReleaseAge` (7 days) blocks freshly-published packages. `trustPolicy: no-downgrade` blocks rollbacks to less-trusted versions. `allowBuilds` allowlist restricts which packages may run postinstall scripts. `common:check:lockfile-age` mise task enforces the 7-day policy independently of pnpm. |
| **Go modules** | **Mend SCA** via `cd.yml` ‚Äî covers Go lockfiles. **Dependabot** ‚Äî configured for `/packages/controller`. **govulncheck** ‚Äî runs on PRs and daily via `mise run scan`, checks compiled call graph against the Go vuln DB. Remediation: bump `go.mod` + `go mod tidy`. | Pinned by `go.sum` checksum database. Go module proxy verifies checksums against the public sumdb. |
| **Container base images** | **Trivy** ‚Äî one `<scope>:scan:trivy` task per image. `mise run scan` builds every image and **fails on fixable findings** (PR gate; `ignore-unfixed` via `trivy.yaml`, so only findings with an available fix gate). `cd.yml`'s `container-scan` runs the same tasks with `--json --tag=<commit-sha>` to scan the image the workflow built and pushed for the current commit (every build publishes a `<sha>` tag ‚Äî amd64-only except `v*` release builds, which are multi-arch; on `main`/`v*` pushes the same build additionally gets the semver/`latest` release tags) ‚Äî and auto-files one GitHub issue per CVE. Remediation: bump the base/deps, rebuild and push. Findings whose fix exists upstream but isn't in a consumable release yet are **left as open issues, not suppressed** ‚Äî label them `blocked-upstream` (`gh issue edit <n> --add-label blocked-upstream`); the gate stays red until the bump lands. | Red Hat Hardened Images (HI, `registry.access.redhat.com/hi/`) pinned by major.minor tag. Keycloak pinned by digest. `gh` built from source with a pinned Go toolchain + bumped `in-toto` (its release binaries lag both the Go security line and that dep). Images sourced from trusted registries (Red Hat, Quay.io). |
| **Third-party container runtime images** | **Not scanned by us.** [Red Hat Hardened Images](https://images.redhat.com/) used wherever possible (`postgresql`, `valkey`, `curl`). Envoy proxy uses an official distroless image. Some utility-only images are from other sources (`quay.io/adorsys/keycloak-config-cli`, `alpine/k8s`). | Tag-pinned in `values.yaml`. Sourced from trusted registries (Red Hat, `registry.k8s.io`). Manual bump only. |
| **GitHub Actions** | **Dependabot** ‚Äî configured for `github-actions` ecosystem, weekly. Auto-files PRs for advisory-affected actions. | Pinned by full commit SHA with version comment (e.g. `actions/checkout@<sha> # v6.0.2`). Tag-only references not used ‚Äî immune to tag rewriting. |
| **mise dev tools** | **Not scanned.** At time of writing, Dependabot and Mend both have open tickets for adding support. | `mise.lock` pins resolved versions with per-platform SHA-256 checksums. |
